# Консольный калькулятор с плагинами

## Постановка задачи (из условия)

Создать консольное приложение `calc.exe`, которое:
- Вычисляет математические выражения с операциями `+ – * / ( )` и возведением в степень `^`
- При запуске автоматически сканирует папку `./plugins/` и загружает все `.dll`
- Каждая DLL содержит ровно одну математическую функцию
- Имя файла DLL **не** участвует в определении имени функции
- Функции становятся доступны сразу после запуска без пересборки основного приложения
- Примеры корректных выражений:  
  `16 + 4 * (3 - 1)` → `24`  
  `2^4 + sin(90)` → `17`

Обязательно обработать все исключительные ситуации (некорректные DLL, отсутствие папки, ошибки в функциях и т.д.).

## Что реально сделано

- Полноценный парсер выражений с правильными приоритетами и поддержкой `^` (правосторонний)
- Динамическая загрузка плагинов через `LoadLibrary` и `GetProcAddress`
- Автоматический поиск функции `create()` в каждой DLL
- Поддержка любых унарных функций: sin, cos, tan, ln, sqrt, deg, rad и т.д.
- Полная обработка всех ошибок и исключений
- Корректное освобождение памяти и выгрузка DLL при завершении
- Всё собирается через CMake одним кликом

## Как собрать проект

```bash
mkdir build && cd build
cmake ..
cmake --build . --config Release 
```

## Как работают плагины 

1. Программа при старте заходит в папку ./plugins/
2. Берёт каждый .dll
3. Вызывает LoadLibraryA() → получает хэндл библиотеки
4. Через GetProcAddress() ищет функцию с именем "create"
5. Если нашла — вызывает её → получает указатель на объект, реализующий интерфейс IFunction
6. Сохраняет указатель и хэндл DLL
7. Добавляет функцию в список доступных
8. При выходе из программы — сначала удаляет все объекты, потом FreeLibrary() для каждой DLL
